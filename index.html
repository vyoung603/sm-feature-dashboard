What Are People Doing When They Open the App?

  • ~22K users/week open the app (trending up from ~19K in Sep to ~23K in Jan-Feb). This is the denominator for all feature penetration.
  • Modules are the primary surface. ~20-25K users/week view modules, ~15-22K view the Prep Module. These are the most-seen in-app experiences.
  • Goal setting is the most-used engagement action at ~1.5-2.7K creators/week (declining from 14% to 6.7% of app openers), followed by reflections (~490 savers/week, 2.2%), journal (~450/week, 2.0%), and mood (~340/week, 1.5%).
  • All engagement features are declining even as app opens grow. The app is getting more users but fewer are engaging with goals, reflections, mood, or journal. For reference, AI Coach text conversations (~800/week, 3.5%) now sit between goals and reflections in raw users.

The Big Picture: Where Goal Setting & Reflections Sit in the App

  • ~22K users/week open the app (growing from ~19K in Sep to ~23K in Feb). This is the base for all penetration calculations.
  • Goal setting reaches ~6.7% of current weekly app openers (~1.5K of ~23K). Period average was 9.5%, but it's declining. Still the most-used engagement feature by raw count.
  • Daily reflections reach only ~2.2% of app openers (~490/week). This is a niche feature with a small but loyal base.
  • For comparison: Prep Module reaches ~96% of app openers, Module Viewed ~109%. Goals and reflections are low-penetration features relative to what users actually interact with in the app.
  • AI Coach text conversations (~800/week, 3.5%) now sit between goals and reflections in raw users as a reference point for scale.

Goal Setting: High Volume Relative to Engagement Features, but Zero Stickiness

  • At ~1.5K users/week (current), goal creation is still the most-used engagement feature by raw count -- larger than reflections, mood, journal, or coach conversations. But the lead is shrinking.
  • But retention is essentially 0% (0.9% D1-7). Users create a goal and never come back to create another. It's a one-and-done interaction.
  • The funnel from "Goals Module Viewed" (80K) to completion (658) converts at 0.8%. Awareness is massive. Follow-through is near zero.
  • Volume dropped ~35% from Sep peak. Combined with zero retention, this suggests early novelty wearing off with no habit loop to sustain it.

Daily Reflections: Tiny Base, Genuine Habit

  • Only ~490 users/week save reflections -- 2.2% of app openers. This is a very small audience.
  • But those who do it, keep doing it: 39% return within a week, 22% still active at D60-90. This is real habit formation.
  • The question isn't "is reflection retention good?" (it is). The question is "does this tiny audience justify the feature's footprint?"
  • Biggest funnel leak: 59% of users who open the reflection screen never begin (41% convert to Begin). But the goal funnel is far worse -- 99.2% drop off at step 1.

Questions to Investigate

  • Is goal creation designed to be one-time? If so, 0% retention is fine -- track edits/engagement instead. If users should set multiple goals, the product needs a re-engagement loop.
  • Something changed in mid-December with goal tracking (Goal Saved appeared, Add Goal jumped 5x). New feature or new instrumentation?
  • What's blocking the reflection "Open to Begin" conversion? 59% drop off before starting. Is the prompt wrong? Is the UI friction too high?
  • Why is AI Coach activation so low? 27K see it, 700 engage. Is it the positioning, the CTA, the first message, or user intent mismatch?
  • Are goal/reflection/coach users the same people? A cohort overlap analysis would reveal if these are complementary features or competing for the same small wellness-curious audience.
  • Platform property is "(none)" for all events. Can't segment iOS vs Android vs Web. This is a tracking gap that should be fixed.
  • "Active Users" numbers (~200K-1M/week) vs "App Opened" (~22K/week). The active user count likely includes web/marketing traffic. Clarify what "active" means in this project -- it inflates all penetration metrics if used as the denominator.

Claims Flagged as Incorrect or Misleading

  • "9.5% penetration" uses the period average. Current week is 6.7% (1,533 / 23K) -- a 44% decline from the Sep peak. The average masks the trajectory.
  • "70% reflection Open-to-Begin dropoff" is wrong. Actual dropoff is 59% (41% convert to Begin). More importantly, the goal funnel has a 99.2% dropoff at step 1 (Module Viewed to Add Goal) -- 17x worse. The dashboard buried the bigger leak.
  • Goal funnel numbers are logically inconsistent. Goal Created (88) < Goal Saved (658). You can't save a goal you didn't create. This points to instrumentation issues or incorrect funnel step ordering. Treat goal funnel data with caution.
  • "Coach would dwarf all features at 5-10%" -- math is correct but speculative. No coach retention data exists yet. Noted for reference, not a driver of this analysis.

Claims Confirmed as Accurate

  • Goal retention is essentially 0%. Confirmed. 0.9% D1-7 out of 53,819 users = fewer than 500 users ever return. This is zero in any practical product sense.
  • Reflection retention (39% D1-7, 22% D60-90) is genuinely strong. Confirmed. Unusual for an optional feature in healthcare. The curve shape (gentle degradation, not a cliff) suggests a real retained cohort.
  • AI Coach already exceeds reflections in raw users. Confirmed. Coach text (~800/wk) > Reflection Save (~356/wk recent). Noted as a scale reference point.

The Case for Freezing Goal Setting

  • The December experiment already ran. A UI update drove more micro-interactions (Add Goal +5x, Goal Edit +4x) but failed to reverse the adoption decline or improve retention. The investment was tested and did not change the trajectory.
  • This is not a UX problem -- it's a user motivation problem. 80K users viewed the Goals module. 655 tapped Add Goal. Better design doesn't fix a 99.2% disinterest rate. Users see it, understand it, and choose not to engage.
  • At current trajectory, Goal Created drops below 1,000 weekly users by summer 2026. Investing dev time in a feature approaching single-digit-hundreds of users has steep opportunity cost.
  • Opportunity cost is real. Every sprint on goal setting UI is a sprint not spent on features with growing user demand. The return on engineering time is higher elsewhere.

The Case for Maintaining (Not Investing In) Reflections

  • The retention is real. 39% D1-7, 22% D60-90 -- this is a genuine habit for those who adopt it. Shutting it down would alienate a loyal (if small) user base.
  • But the base is too small and declining to justify new feature work. ~530 weekly savers declining toward ~350. Even the retained cohort is only ~250-350 users at D60-90 from monthly cohorts.
  • If you want to test growth, try surfacing/prompting before rebuilding UI. A notification or in-app prompt experiment is a 1-2 sprint test, not a quarter of dev work. The bottleneck might be discovery, not quality.
  • Monitor for natural cannibalization. As other in-app features evolve, watch whether reflections decline further or stabilize. If another feature absorbs the use case, the decline is healthy platform evolution.

Critical Caveat: The One Thing That Could Reverse This

  • We do not have data on whether goal setting or reflections correlate with session retention or therapy outcomes.
  • If users who set goals attend 20% more sessions, or reflection users have lower churn, the in-app retention numbers are irrelevant -- the feature works through the therapy relationship, not through the app.
  • This analysis must be run before making a final decision. It requires joining Amplitude behavioral data with session/billing data. Without it, we are optimizing for app engagement metrics when the real value may be clinical.
  • Similarly, if therapists are actively referencing client goals in sessions, the client-side retention metric is misleading -- the feature is being consumed via the provider, not the app.

Missing Data: What We'd Need for Full Confidence

  • Correlation with client retention. Do goal/reflection users stay longer or engage more broadly? This is the #1 missing analysis and could reverse the recommendation.
  • Outcome measures. PHQ-9/GAD-7 score changes correlated with feature usage -- the ultimate test for whether these features matter clinically.
  • Cohort overlap. Are goal/reflection/mood/journal users the same people? If so, these features compete for a small wellness-curious segment rather than serving distinct needs.
  • User tenure segmentation. Are these features used by new clients exploring the app, or established clients deepening engagement? Different answers imply different strategies.
  • Feature discovery rates. What % of app openers even know goals and reflections exist? Low adoption might be a discovery problem, not a value problem. Cheap to test with better surfacing before any UI investment.
  • The "Active Users" denominator. 200K-1M "active" users vs 22K app openers -- the real engaged client base is likely closer to 22K. All penetration math depends on which number you use.
